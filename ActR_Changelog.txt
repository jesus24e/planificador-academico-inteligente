ActR_Changelog
==============

Rama: integracion del boton "Nueva tarea" tras el merge del equipo.
Objetivo de esta entrega: diagnosticar y corregir por que el boton
"Nueva tarea" / "Anadir tarea" no funcionaba.


===========================================================================
DIAGNOSTICO - Que estaba fallando
===========================================================================

[CAUSA PRINCIPAL]
El boton "Anadir tarea" de la pantalla de Inicio SI abria el dialogo,
pero el boton "Guardar" dentro del dialogo no hacia nada.
En lib/ui/widgets/home/aniadirTarea.dart el onPressed del boton Guardar
tenia el cuerpo vacio:

    onPressed: () {
      //  guardar datos
    },

Es decir, el companero construyo toda la interfaz del formulario pero
dejo pendiente la logica de guardado. Al pulsar "Guardar" no se insertaba
nada en la base de datos y el dialogo ni siquiera se cerraba.

[PROBLEMAS SECUNDARIOS DETECTADOS]
1. El dialogo usaba un TextField libre para "materia". Como existe una
   llave foranea actividades.materia -> materias.nombre, guardar una
   materia inexistente habria provocado un error de constraint.
2. El dropdown de "tipo" solo tenia 3 opciones (tarea, examen, proyecto);
   faltaba "actividad", definida como una de las 4 opciones oficiales.
3. Quedaron DOS dialogos para la misma funcion: aniadirTarea.dart (del
   companero, conectado pero incompleto) y nueva_tarea_dialog.dart (de
   una iteracion previa, completo pero ya sin uso tras el merge).
4. El boton "Nueva tarea" de la pestana "Mis tareas" seguia deshabilitado
   (onPressed: null) desde una iteracion anterior.
5. Warning menor: faltaba la anotacion @override sobre el metodo build
   en home_screen.dart.

RESULTADO
===========================================================================

- El boton "Anadir tarea" (Inicio) ahora guarda correctamente la tarea
  en la base de datos.
- El boton "Nueva tarea" (pestana Mis tareas) quedo reactivado y abre el
  mismo dialogo; al guardar, la lista de tareas se refresca sola.
- Ambos botones validan que la materia exista antes de guardar, evitando
  errores de integridad con la base de datos.
- Ya no hay codigo duplicado activo: existe un unico dialogo oficial.


Notas adicionales
-----------------
- No se modifico la base de datos ni las entidades en esta entrega; el
  problema era puramente de logica de UI y de cableado tras el merge.
- Para que las tareas creadas desde Inicio aparezcan en esa misma
  pantalla sin reiniciar la app, en una iteracion futura habria que
  conectar home_screen (y priorityList) a la base de datos en lugar de
  mapDateActivity.

Iteracion 5 - Fix: quitar tareas vencidas de la Lista de prioridades
===========================================================================

[16] Archivo: home_screen.dart  (MODIFICADO)
     Ubicacion: lib/ui/screens/home/home_screen.dart
     Proposito:
      - Se agrego el metodo "_quitarVencidasDePrioridad", que se ejecuta
        al cargar la pantalla (dentro de _cargar), antes del
        auto-agregado por calendario.
      - Recorre las tareas que estan en la Lista de prioridades
        (prioridad_estado = 1) y, si su fecha de entrega es anterior al
        dia de hoy, las cambia a prioridad_estado = 0. Asi salen de la
        Lista de prioridades pero se mantienen en "Mis tareas".
      - La comparacion se hace solo por dia (ignora la hora): una tarea
        que vence HOY no se considera vencida, sigue en la lista y
        ademas alimenta el contador "vencen hoy".
      - Se eligio el estado 0 (ninguno) y no el 2 (descartada) para no
        mezclar el significado "el usuario la quito a proposito" con
        "se vencio". Como la fecha ya paso, el auto-agregado por
        calendario tampoco la reincorpora (una fecha pasada nunca cae
        en las 2 semanas futuras del calendario), por lo que el estado
        0 es seguro y no provoca que la tarea reaparezca sola.

Nota
----
- El orden de ejecucion es seguro: primero se quitan las vencidas y
  luego corre el auto-agregado. Aunque ambos metodos recorren la misma
  lista en memoria (cargada una sola vez), una tarea recien vencida
  tiene fecha pasada y nunca cumple la condicion de fecha futura del
  auto-agregado, asi que no hay conflicto entre ambas operaciones.
